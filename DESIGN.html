StuRoom — Database & Feature Design

Tài liệu thiết kế hệ thống · Cập nhật lần cuối: 2026-05-22

Quy tắc bắt buộc: Mọi thay đổi thiết kế phải được cập nhật vào file này TRƯỚC khi viết code hoặc tạo migration. Ghi vào bảng Lịch sử thay đổi.

Bối cảnh dự án

LoạiĐồ án môn học (1 kỳ), nhóm 2–4 người
StackASP.NET Core MVC · EF Core · SQL Server · Cloudinary · Export PDF + Word
Mục tiêuHệ thống hybrid — vừa marketplace tìm phòng cho sinh viên, vừa công cụ quản lý cho chủ trọ

Roles

RoleQuyền chính
AdminQuản lý toàn hệ thống, duyệt nội dung
LandlordQuản lý tòa nhà, phòng, hợp đồng, hoá đơn
TenantTìm phòng, đặt lịch xem, ký hợp đồng, trả tiền, đánh giá

Workflows

Tenant Người thuê

1
Đăng ký tài khoản — chọn role Tenant, xác nhận email. Không cần Admin duyệt.
2
Tìm phòng — lọc theo giá, địa điểm (Tỉnh/Quận/Phường), diện tích, tiện ích, đang tìm ghép.
3
Gửi lịch hẹn xem phòng — đề xuất giờ + ghi chú → Landlord confirm/đổi giờ → nhận email xác nhận.
4
Đặt phòng — sau khi xem, bấm "Đặt phòng" trên app, điền ngày dự kiến vào ở + lời nhắn → tạo BookingRequest (Pending).
5
Ký hợp đồng — Landlord duyệt BookingRequest → tạo Contract → Tenant download PDF/Word, ký tay, trả lại bản scan.
6
Thanh toán hàng tháng — nhận thông báo hoá đơn mới → xem chi tiết → Landlord ghi nhận thanh toán.
7
Tìm người ghép (tuỳ chọn) — bật SeekingRoommates → listing hiển thị "đang tìm ghép X chỗ" → người khác đăng ký vào ContractMembers.
8
Đánh giá — sau khi hợp đồng kết thúc, viết review + rating cho phòng.

Landlord Chủ trọ

1
Đăng ký tài khoản — chọn role Landlord → Admin duyệt → mới được đăng phòng.
2
Tạo tòa nhà → tạo phòng — upload ảnh (Cloudinary), chọn tiện ích, cấu hình phí (Building/Room level).
3
Xử lý lịch hẹn xem — nhận thông báo → confirm hoặc đổi giờ → tenant nhận email.
4
Duyệt BookingRequest — xem thông tin tenant + ngày dự kiến → Approve (tạo Contract) hoặc Reject.
5
Quản lý hợp đồng — tạo/gia hạn/chấm dứt hợp đồng, export PDF + Word.
6
Tạo hoá đơn hàng tháng — nhập chỉ số điện/nước → hệ thống tính tự động → gửi cho tenant.
7
Ghi nhận thanh toán — xác nhận tenant đã trả → đổi trạng thái Invoice → Paid.
8
Dashboard — doanh thu theo tháng, tỷ lệ lấp đầy phòng, danh sách hoá đơn quá hạn.

Admin Quản trị

1
Duyệt tài khoản Landlord — xem danh sách Landlord chờ duyệt → Approve/Reject. Tenant không cần duyệt.
2
Quản lý danh mục Amenities — thêm/sửa/xoá tiện ích dùng chung toàn hệ thống.
3
Kiểm duyệt review — ẩn review vi phạm (IsApproved = false).
4
Dashboard tổng platform — tổng số phòng, users, Landlord active, doanh thu toàn hệ thống.
📌 RULE — Landlord approval: Landlord mới đăng ký có IsApproved = false. Chưa được duyệt thì không tạo được Building/Room.

Database Schema

string — bắt buộc, không được null
string? — nullable, có thể để trống
FK → — khoá ngoại liên kết bảng khác
int PK — khoá chính, tự tăng

Users

AspNetUsers (extends IdentityUser → ApplicationUser)
  + FullName       string
  + DateOfBirth    DateTime?
  + AvatarUrl      string?
  + StudentId      string?    -- chỉ dành cho Tenant
  + IsApproved     bool       -- chỉ dùng cho Landlord, default = false khi đăng ký

Tài sản

Buildings
  Id            int PK
  LandlordId    FK → AspNetUsers
  Name          string
  Address       string
  Province      string
  District      string
  Ward          string
  Description   string?
  CreatedAt     DateTime

Rooms
  Id            int PK
  BuildingId    FK → Buildings
  RoomNumber    string
  FloorNumber   int?
  Area          decimal   -- m²
  Capacity      int?      -- null = không giới hạn số người thuê
  Description   string?
  Status        enum: Available | Occupied | Maintenance | Inactive
  CreatedAt     DateTime

RoomImages
  Id                  int PK
  RoomId              FK → Rooms
  ImageUrl            string   -- Cloudinary URL
  CloudinaryPublicId  string
  IsPrimary           bool
  SortOrder           int

Amenities  -- danh mục tiện ích (WiFi, Điều hoà, Máy giặt, Bãi xe…)
  Id         int PK
  Name       string
  IconClass  string   -- CSS icon class

RoomAmenities  -- many-to-many, composite PK (RoomId, AmenityId)
  RoomId     FK → Rooms
  AmenityId  FK → Amenities
📌 RULE — Room.Status: Tự động cập nhật theo Contract active — không toggle thủ công. Có Contract.Status = Active → Occupied; không có → Available.

Cấu hình phí Điểm khác biệt

FeeConfigs
  Id            int PK
  BuildingId    FK → Buildings  (nullable)
  RoomId        FK → Rooms      (nullable) ← override cấp phòng
  Name          string          -- VD: "Tiền điện", "Phí xe máy"
  FeeCategory   enum: Rent | Electricity | Water | Internet | Parking | Other
  CalcType      enum: Fixed | PerUnit
  UnitPrice     decimal
  Unit          string          -- "tháng", "kWh", "m³", "xe"…
  IsActive      bool
  SortOrder     int
📌 RULE — Ưu tiên khi tạo hoá đơn: RoomId-level override BuildingId-level cùng FeeCategory. Lấy tất cả FeeConfig IsActive = true áp dụng cho phòng đó.

Lịch hẹn xem phòng

ViewingRequests
  Id              int PK
  RoomId          FK → Rooms
  TenantId        FK → AspNetUsers
  ProposedTime    DateTime    -- tenant đề xuất
  ConfirmedTime   DateTime?   -- chủ đổi giờ (nullable)
  Status          enum: Pending | Confirmed | Rescheduled | Cancelled | Completed
  TenantNote      string?
  LandlordNote    string?
  CreatedAt       DateTime
📧 RULE — Email: Gửi email cho Tenant khi Status chuyển sang Confirmed hoặc Rescheduled.

Booking Requests Đặt phòng

BookingRequests   -- cầu nối giữa "xem phòng" và "ký hợp đồng"
  Id                 int PK
  RoomId             FK → Rooms
  TenantId           FK → AspNetUsers
  ViewingRequestId   FK → ViewingRequests  (nullable — có thể đặt không qua lịch xem)
  DesiredMoveIn      DateTime    -- ngày dự kiến vào ở
  Message            string?     -- lời nhắn cho landlord
  Status             enum: Pending | Approved | Rejected
  RejectionReason    string?
  ContractId         FK → Contracts  (nullable — được set khi Landlord Approve)
  CreatedAt          DateTime
📌 RULE — Approve flow: Landlord Approve → hệ thống tự tạo Contract (Status = Pending) → gán ContractId vào BookingRequest → thông báo Tenant.
📌 RULE — Giới hạn: Một phòng chỉ có thể có 1 BookingRequest Status = Pending cùng lúc. Tenant không thể gửi 2 request cho cùng 1 phòng.

Hợp đồng

Contracts
  Id                 int PK
  RoomId             FK → Rooms
  TenantId           FK → AspNetUsers   -- người ký HĐ (primary)
  StartDate          DateTime
  EndDate            DateTime?           -- null = open-ended
  ActualEndDate      DateTime?           -- ngày thực tế kết thúc
  Status             enum: Pending | Active | Expired | Terminated
  DepositAmount      decimal
  MonthlyRent        decimal             -- tổng tiền phòng, không tự đổi khi ghép
  SeekingRoommates   bool               -- đang tìm người ghép
  DesiredOccupancy   int?               -- muốn ở bao nhiêu người tổng (null = không giới hạn)
  GeneratedDocUrl    string?
  TerminationReason  string?
  Notes              string?
  CreatedAt          DateTime

ContractMembers   -- người ở ghép, đăng ký vào hệ thống (không ký HĐ riêng)
  Id           int PK
  ContractId   FK → Contracts
  TenantId     FK → AspNetUsers
  JoinDate     DateTime
  LeaveDate    DateTime?   -- null = đang ở
  Note         string?
👥 RULE — Occupancy: Tổng người ở = 1 (primary) + ContractMembers có LeaveDate = null. Phải ≤ Room.Capacity (nếu có) và ≤ Contract.DesiredOccupancy (nếu có).
💰 RULE — Tiền phòng: MonthlyRent là giá tổng phòng, không tự thay đổi khi người ghép vào/ra. Việc chia tiền nội bộ do các tenant thoả thuận với landlord.
🔍 RULE — Tìm ghép: Hiển thị "đang tìm ghép" khi SeekingRoommates = true. Số chỗ còn: DesiredOccupancy - current occupants.
📌 RULE — Lịch sử: Không xoá contract cũ, chỉ đổi Status. Khi member rời đi, set LeaveDate — không xoá dòng.
📄 RULE — Template: Chỉ dùng template mặc định của app (không cho upload custom). Xuất cả PDF lẫn Word.

Hoá đơn & Thanh toán

Invoices
  Id            int PK
  ContractId    FK → Contracts
  BillingYear   int
  BillingMonth  int      -- 1–12
  TotalAmount   decimal
  Status        enum: Draft | Sent | Paid | Overdue | Cancelled
  DueDate       DateTime
  Notes         string?
  CreatedAt     DateTime

InvoiceItems
  Id                int PK
  InvoiceId         FK → Invoices
  FeeConfigId       FK → FeeConfigs  (nullable — có thể là dòng thủ công)
  Description       string
  Quantity          decimal?  -- null nếu Fixed
  UnitPrice         decimal
  Amount            decimal
  PreviousReading   decimal?  -- cho PerUnit (điện/nước)
  CurrentReading    decimal?

Payments
  Id              int PK
  InvoiceId       FK → Invoices
  Amount          decimal
  PaymentDate     DateTime
  Method          enum: Cash | BankTransfer | MoMo | VNPay
  TransactionRef  string?
  Notes           string?
  RecordedById    FK → AspNetUsers
  CreatedAt       DateTime
🧮 RULE — Tính tiền: Amount = Quantity × UnitPrice (PerUnit) hoặc UnitPrice (Fixed).
➗ RULE — Phòng ghép: Hoá đơn điện/nước chia đều tự động 1/N người đang active trong phòng.

Đánh giá & Thông báo

RoomReviews
  Id           int PK
  RoomId       FK → Rooms
  ReviewerId   FK → AspNetUsers
  ContractId   FK → Contracts   -- xác minh đã từng ở phòng đó
  Rating       int              -- 1–5
  Content      string?
  IsApproved   bool             -- admin/landlord có thể ẩn
  CreatedAt    DateTime

Notifications
  Id                  int PK
  UserId              FK → AspNetUsers
  Type                enum: ViewingConfirmed | ViewingRescheduled | ViewingCancelled
                             | InvoiceDue | ContractExpiring | ContractSigned
                             | NewViewingRequest | PaymentReceived | NewReview
  Title               string
  Content             string
  IsRead              bool
  RelatedEntityType   string?   -- "Contract", "Invoice", "ViewingRequest"…
  RelatedEntityId     int?
  CreatedAt           DateTime
